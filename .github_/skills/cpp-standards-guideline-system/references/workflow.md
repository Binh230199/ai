# Workflow

Use this workflow for standards-to-guideline synthesis.

## Phase 1: Intake

- capture the standard name
- capture the source document path or file URI
- define the output file path
- define the artifact root

## Phase 2: Ingestion

- parse the PDF to markdown or text
- record extraction warnings and limitations

## Phase 3: Rule Normalization

- convert source content into structured rule records
- separate source wording from engineering interpretation

## Phase 4: Interaction Mapping

- identify reinforcing, overlapping, prerequisite, and conflict-risk relationships
- build the rule graph artifact

## Phase 5: Core Guideline Synthesis

- cluster rules into core families
- derive the preferred canonical pattern for each family
- explain why the canonical form is the safest default

## Phase 6: Verification

- check artifact completeness
- check final markdown sections
- check relationship coverage and reasoning quality
