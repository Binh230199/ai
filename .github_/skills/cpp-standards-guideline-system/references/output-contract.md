# Output Contract

The final markdown file such as `cpp-core-autosar.md` or `cpp-core-misra.md` must include the sections below.

## Required Sections

- `# Title`
- `## Source and Scope`
- `## Analysis Method`
- `## Core Rule Families`
- `## Preferred Canonical Patterns`
- `## Rule Interaction Map`
- `## Fix-Order Strategy`
- `## Review Checklist`
- `## Rule Index`

## Required Qualities

- the document is original and paraphrased
- it does not merely mirror the source document order
- it identifies multi-rule canonical patterns
- it explains conflict-risk and fix-order traps
- it points back to rule identifiers for traceability
