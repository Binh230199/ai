# Final Guideline Output Rules

- The final guide must be a synthesized engineering document, not a raw restatement of the standard.
- Organize guidance by core rule family, not by source document order.
- Each core rule family should include the preferred canonical pattern, why it is preferred, and what other rules it helps satisfy.
- Include anti-patterns and fix-order traps where relevant.
- Keep code examples short and original.
- Do not copy long text from the standard.

## Required Sections

- Source and scope
- Analysis method
- Core rule families
- Preferred canonical patterns
- Rule interaction map
- Fix-order strategy
- Review checklist
- Rule index or source mapping appendix
