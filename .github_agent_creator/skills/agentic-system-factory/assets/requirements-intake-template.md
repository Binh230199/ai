# Requirements Intake Template

## Objective

- What system should be designed?
- What job should it complete end-to-end?

## Scope

- What repositories, modules, services, or workflows are in scope?
- What is explicitly out of scope?

## Inputs

- What data, APIs, MCP servers, scripts, or artifacts are available?

## Constraints

- Security and approval rules
- Runtime and environment constraints
- Build, test, and deployment constraints
- Cost, latency, or quota constraints

## Completion Criteria

- What measurable conditions define success?

## Deliverables

- What should the system produce for users or operators?

## Unknowns

- What information is still missing but materially affects the design?
